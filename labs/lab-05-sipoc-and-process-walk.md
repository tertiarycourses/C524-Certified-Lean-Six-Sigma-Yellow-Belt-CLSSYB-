# Lab 05: SIPOC and Process Walk

## Domain

Process and Value Stream Mapping

## Objective

Create a SIPOC diagram and validate it with process walk observations.

## Tasks

1. Define the process start point.
2. Define the process end point.
3. Identify suppliers.
4. Identify inputs.
5. List 5 to 7 high-level process steps.
6. Identify outputs.
7. Identify customers.
8. Perform a process walk or simulation.
9. Update the SIPOC based on what you observed.
10. Mark questions or assumptions that require follow-up.

## Deliverable

Submit a SIPOC diagram and process walk notes.

## Review Questions

1. How does SIPOC help define project scope?
2. Why should process maps be validated?
3. What did the process walk reveal that was not obvious?
4. Which input quality issues affect the process most?
